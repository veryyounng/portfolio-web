TicTacToe 미니 게임

TicTacToe 미니 게임

승/무/패 카운트와 리셋 기능을 제공하는 미니 게임입니다.

행렬 연산과 탐색 알고리즘을 적용하여 논리적 사고력과 알고리즘 설계를 강조했습니다.

(1) 행렬/벡터 기반 보드 표현

  • 보드는 2차원 배열(3×3 행렬)로 표현 가능하지만, 성능과 단순화를 위해 1차원 배열(길이 9)을 사용했습니다.
  • 인덱스를 좌표계(행, 열)로 변환하는 과정에서 선형대수학적 사고 (행렬 인덱스 변환, 좌표 매핑)를 적용했습니다.
  • board[i][j] ↔ board[i*3 + j]

(2) 승리 조건 탐색

가로, 세로, 대각선의 승리 조건을 벡터의 합/곱 개념으로 일반화하여, "세 칸의 합이 모두 같은 플레이어일 때 승리"라는 규칙을 행렬 연산 → 탐색 알고리즘 형태로 해석했습니다.

(3) AI 알고리즘 (Minimax)

플레이어의 모든 선택지를 재귀적으로 탐색하며 최적의 수를 찾는 Minimax 알고리즘을 적용했습니다. 이는 수학적으로 트리 탐색(DFS)과 게임이론의 minimax 함수 개념을 활용한 것입니다.

휴리스틱 함수로 승리 시 +10, 패배 시 −10, 무승부는 0을 부여하여 점수를 계산하는 Utility Function을 정의했습니다.

방향 전환 공룡 게임

방향 전환 공룡 게임

Docker 컨테이너화 & AWS EC2 클라우드 배포, Python, JavaScript

FastAPI 기반 백엔드를 Docker로 컨테이너화하고, 동일한 환경을 AWS EC2(Ubuntu 22.04) 인스턴스에 배포했습니다.

방향 전환 수학과 상태 전이 원리를 체화하기 위해 직접 구현한 2D 시뮬레이션 미니 게임입니다.

개발 동기

백준 3190 ‘뱀’ 문제의 유사 로직을 직접 구현하면서, 방향 전환의 수학적 개념과 상태 전이 구조를 게임 로직 속에 녹여냈습니다.

설계와 수학적 원리

(1) 보드 모델링 (행렬/벡터)
  • 개념적으로는 2D 행렬 N×N, 구현은 단순화를 위해 1D 인덱스로 투영
  • 좌표 → 인덱스 변환 수식: idx = y * N + x
(2) 방향 벡터 & 회전
  • 이동 수식: p_next = p_current + d
  • 회전은 두 가지 방식으로 구현 가능 - 행렬 회전: d' = R(θ)·d - 테이블 회전: Direction[i] → Direction[(i±1) mod 4]
  • 충돌 판정: 벽이나 자기 몸집 좌표를 Set으로 저장하고, p_next가 해당 집합에 포함되면 충돌로 처리
